# [PRD] 전문 사진가를 위한 PWA 노출계: AnalogMeter

## 1. 프로젝트 개요
* **제품명:** AnalogMeter
* **저장소:** `noveler17/analog-meter` (GitHub)
* **플랫폼:** PWA (Progressive Web App)
* **핵심 철학:** 전문 수동 카메라 사용자를 위한 고성능 외장 노출계. 1/3 스탑 단위의 정밀 제어와 존 시스템(Zone System) 시각화를 제공하며, 녹색 테마의 아날로그 감성을 유지함.

---

## 2. UI/UX 레이아웃 정의

화면은 세로형(Portrait)을 기준으로 하며, 다양한 크기의 스마트폰 환경을 케어하는 반응형 레이아웃을 적용한다.

### 2.1. 상단: 카메라 및 측정 영역 (Fixed Top)
* **라이브 뷰:** 실시간 카메라 프리뷰 영역.
* **가이드 박스:** 초점 거리 다이얼 값에 연동되어 실시간으로 크기가 변하는 사각형.
* **존 시스템 UI:**
    * **Z-SYS Toggle:** 모드 On/Off 스위치.
    * **Zone Bar:** 우측에 위치하며 Zone III ~ VII 단계를 **무채색 그라데이션**으로 시각화.
    * **Spot Meter:** 화면 탭 시 원형 인디케이터가 생성되며, 해당 지점의 존 상태를 색상으로 즉시 매핑.
* **정보 표시:** 현재 측정된 실시간 EV 값을 가이드 박스 상단에 표시.

### 2.2. 중단: 조합 쌍 영역 (Variable Center)
* **Combination Pairs 리스트:** 현재 EV 값에 부합하는 모든 조리개(f) 및 셔터 스피드(ss) 조합을 출력.
* **가변 레이아웃:** 설정에 따라 리스트 영역이 확장되어 더 많은 조합을 한눈에 볼 수 있도록 설계.
* **시각화 규칙 (Highlight Logic):**
    * **설정창:** 사용자가 '우선순위 F 범위'와 '우선순위 SS 범위'를 직접 지정할 수 있는 오버레이 팝업 제공.
    * **초록색 글씨:** 설정된 F 범위 또는 SS 범위 중 **하나만 만족**하는 조합.
    * **초록색 테두리:** 설정된 F 범위와 SS 범위 **둘 다 만족**하는 조합.

### 2.3. 하단: 제어 및 설정 영역 (Fixed Bottom)
* **MEASURE 버튼:** 현재 노출값을 로그로 확정 저장하는 대형 버튼.
* **초점 거리 다이얼:** 기기 기본 환산치(1x) ~ 5배 줌 영역까지 횡스크롤 다이얼로 조절.
* **ISO 다이얼:** **ISO 25**부터 시작하여 1/3 스탑 단위로 스냅(Snap)되는 다이얼.
* **노출 보정(EC) 다이얼:** -3 ~ +3 EV 범위 (1/3 스탑 단위).
* **프리셋 섹션:** `SAVE PRESET`, `LOAD PRESETS` 버튼 및 최근 저장된 프리셋 슬롯.
* **GitHub 버튼:** 우측 상단 또는 하단 여백에 위치하며, **GitHub 로고는 흰색**으로 처리하여 정체성 유지.

## 3. 기능 및 기술 명세

### 3.1. 하드웨어 대응 (Device DB)
* iOS(iPhone 13~17 시리즈), Galaxy(S22~S26 시리즈)의 최신 5세대 모델 사양 내장.
* 35mm 환산 초점 거리, 고정 조리개, 센서 규격을 기준으로 모든 계산 수행.

### 3.2. 정밀도 및 범위
* **ISO:** 25 ~ 12800+ (1/3 스탑 단위).
* **조리개:** f/1.2 ~ f/22 (현실적 수동 렌즈 범위).
* **셔터 스피드:** 30s ~ 1/8000s (결과값 출력 전용).
* **노출 조절:** 모든 다이얼은 사용자가 선택 가능한 1/3 스탑 단위로만 조작 및 표시됨.

### 3.3. 성능 최적화
* **Non-blocking:** 픽셀 휘도 분석 루프는 **Web Worker**에서 수행하여 60fps 인터페이스 유지.
* **OffscreenCanvas:** 렌더링 성능을 저하시키지 않도록 백그라운드 캔버스 활용.
* **제스처 제어:** `touch-action: manipulation` 및 뷰포트 메타 태그 설정을 통해 **더블 탭 줌 방지**.

## 4. GitHub 배포 파이프라인 (CI/CD)

배포 시 기존 서비스에 문제가 생기지 않도록 현대적인 Actions 워크플로우를 구성한다.

### 4.1. 브랜치 전략
* **main:** 프로덕션 코드 브랜치. 모든 변경사항은 PR(Pull Request)을 거쳐 머지된다.
* **gh-pages:** 빌드된 정적 결과물만 포함하는 배포 전용 브랜치 (Actions에 의해 자동 관리).

### 4.2. 배포 파이프라인 (`.github/workflows/deploy.yml`)
* **검증 단계:** 빌드 전 Lint 체크 및 테스트 수행. 실패 시 배포 중단.
* **빌드 단계:** `pnpm build`를 통해 `dist` 폴더 생성.
* **안전 배포:** * GitHub Actions의 `Upload-pages-artifact`와 `Deploy-pages`를 사용하여 **Artifact 기반 배포** 수행.
    * 배포 실패 시 이전 버전의 아티팩트가 유지되도록 구성하여 무중단 서비스 보장.
* **문제 제보:** 앱 내 설정 메뉴 또는 GitHub 버튼 클릭 시 `noveler17/analog-meter/issues`로 연결.
